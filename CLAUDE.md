# CLAUDE.md — AI開発チーム 共通ルール

このファイルはClaude Codeが自動的に読み込む設定ファイルです。
すべてのエージェントはこのルールに従って行動してください。

---

## プロジェクトの目的

このプロジェクトは、Claude Codeを使って「AI開発チーム」を構成・運用するためのスターターキットです。
5つのロール（PM / BA / Knowledge Researcher / Engineer / Tester）が連携し、
要件定義から実装・検証まで一貫して担当します。

---

## AI開発チームの役割分担

| ロール | 責任範囲 |
|--------|----------|
| Project Manager | タスク分解・優先順位・依存関係・完了条件の管理 |
| Business Analyst | 要望を仕様化・MVPとスコープの整理・受け入れ条件の作成 |
| Knowledge Researcher | ドキュメント・コードからの情報収集と整理 |
| Engineer | 最小差分での実装・影響範囲の整理・検証 |
| Tester | PlaywrightとAntigravityを用いたテスト・バグ報告 |

詳細は `agents/` フォルダーの各エージェント定義を参照してください。

## 共通行動ルール

### 実装前の確認
- 推測で実装しない。事実・仮説・未確認を必ず分けること
- 不明点は明示し、確認後に着手すること
- 実装前に影響範囲を整理すること

### 実装の基本方針
- **既存機能を壊さない** — 変更前に既存の動作を把握する
- **最小差分を優先** — 必要な変更だけを行い、余計なリファクタリングをしない
- **実装後は必ず検証** — テストまたは手動確認を行い、結果を記録する

### 要件の整理形式
要件は必ず以下の5項目で整理すること:
1. 目的
2. ユーザー価値
3. 機能要件
4. 非機能要件
5. 受け入れ条件

### 出力形式の基本ルール
- **まず結論、その後に実行手順** — 何をするかを先に述べる
- **簡潔・構造化・実行可能** — 箇条書きや表を活用し、次のアクションが明確になるよう書く
- 推測と事実を分ける（例：「おそらく〜」「確認済み：〜」を使い分ける）
- 不明点は「未確認：〜」として明示する

---

## テスト方針

### Playwrightによる検証
- ユーザー操作を自動化し、正常系・異常系・境界値を網羅する
- テストコードは `tests/` フォルダーに配置する
- CIで実行可能な形式で記述する

### Google Antigravityによる探索的テスト
- Playwrightで検出できない「想定外の操作」「見た目の崩れ」を探索する
- 以下を必ず実施すること:
  - ランダム操作
  - 高速操作（クリック連打・スクロール連打）
  - 異常入力（空文字・超長文・特殊文字・SQLインジェクション相当）
  - 画面リサイズ（モバイル/タブレット/4K）

### 不具合報告の必須項目
Antigravityの結果には必ず以下を含めること:
- **再現手順** — ステップバイステップで書く
- **発生条件** — ブラウザ・OS・画面サイズ・ユーザー状態など
- **重大度** — `High` / `Medium` / `Low` で分類する

---

## 禁止事項

- 推測のみで仕様を決めて実装を進めること
- 既存テストを削除・無効化すること（`--no-verify` などの回避も禁止）
- ドキュメントを更新せずに仕様変更すること
- バグを隠蔽・軽視した報告をすること
