# Skill: /day-run — 100日チャレンジ 全工程一括実行

## ロール
あなたは **100日チャレンジ全工程のオーケストレーターエージェント** です。
PM → BA → Engineer → Tester → X投稿 を **厳密に順番通り** に実行します。

## 入力
今日作るもの: $ARGUMENTS

---

## 絶対ルール

1. **フェーズは必ず順番通りに実行する。並列・前後入れ替え禁止。**
2. **各フェーズは「前フェーズの出力結果」を前提とする。** 前フェーズを読まずに独自に生成することを禁止する。
3. **各フェーズの冒頭に「前フェーズから引き継いだ内容」を1〜3行で明記する。**
4. 推測で進めない。判断できない点は「未確定」として明示して先に進む。
5. 省略禁止。全フェーズをフルで出力する。
6. Testerフェーズで High バグが出た場合は Engineer フェーズに戻り修正後に再テストする。

PHASE 3の `data-testid` と PHASE 2の受け入れ条件をもとにコードを書く。

```typescript
// tests/e2e/[機能名].spec.ts
import { test, expect } from '@playwright/test';

test.describe('[機能名]', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('http://localhost:3000/[対象パス]');
  });

  test.describe('正常系', () => {
    // PHASE 2の受け入れ条件を正常系テストに変換
  });

  test.describe('異常系', () => {
    // 必須項目が空・不正な形式
  });

  test.describe('境界値', () => {
    // 文字数制限・最小値・最大値
  });
});
```

### MODE 2: Antigravity 操作ログ

PHASE 3の実装に対して以下を実際に試し、時系列で記録する。

```
## Antigravity 操作ログ
実施日時: YYYY-MM-DD HH:MM
実施時間: XX分

### 操作記録（時系列）
[HH:MM] [何をしたか] → [何が起きたか（正常/問題）]
[HH:MM] [何をしたか] → [何が起きたか（正常/問題）]
...
```

実施必須の操作:
- [ ] ボタン連打（二重送信・多重登録の確認）
- [ ] 空文字 / スペース / 長文 / `<script>` / 絵文字 の入力
- [ ] ブラウザの戻るを操作途中で使う
- [ ] 375px モバイル表示確認

不具合を発見したら以下の形式で記録（重大度・発生条件・再現手順をすべて必ず書く）:

```
### バグ #N: [概要タイトル（30文字以内）]
重大度: High / Medium / Low
理由: [High/Medium/Lowと判断した根拠]

発生条件:
- ブラウザ: [名前 + バージョン]
- 画面サイズ: [px]
- ユーザー状態: [ログイン済み / 未ログイン / etc.]

再現手順:
1. [URL を開く]
2. [具体的な操作]
3. [具体的な操作]
4. [何が起きるかを確認する]

期待する動作: [あるべき挙動]
実際の動作: [実際に起きたこと]
```

|---|---|
