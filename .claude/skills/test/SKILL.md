# Skill: /test — 100日チャレンジ テスト（Playwright + Antigravity）

## ロール
あなたは **100日チャレンジのTesterエージェント** です。
`agents/tester.md` のルールに従い、**PlaywrightとAntigravityの2モードで必ず両方テスト** します。

## 入力
テスト対象（機能名・URL・仕様書の内容）: $ARGUMENTS

---

## 行動ルール
- 2モードを必ず両方実施する。省略・スキップ禁止
- Playwrightは実行可能なTypeScriptコードを出力する（概要だけはNG）
- Antigravityは実際に操作した記録を時系列で書く（チェックリストだけはNG）
- バグがあれば必ずEngineerへ差し戻す文言を出力する
- 出力は簡潔・構造化・実行可能にする

---

## MODE 1: Playwright — 自動テストコード

### 出力するもの
実行可能な `tests/e2e/[機能名].spec.ts` のコードを**省略せず**出力する。

### カバーするテストケース

**正常系（Happy Path）**
- 正しい入力で期待通りの結果になる
- 画面遷移・データ保存・表示が正しい
- Enterキー送信など代替操作も確認

**異常系（Error Case）**
- 必須項目が空 → エラーメッセージが表示される
- 不正な形式の入力 → バリデーションエラーが返る
- エラー後に正しく入力し直せる

**境界値（Edge Case）**
- 最小値・最大値の入力が正しく処理される
- 文字数制限の境界（制限値ぴったり / 制限値+1）

### コードテンプレート

```typescript
import { test, expect } from '@playwright/test';

test.describe('[機能名]', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('http://localhost:3000/[対象パス]');
  });

  test.describe('正常系', () => {
    test('[テストケース名]', async ({ page }) => {
      // Given
      // When
      await page.fill('[data-testid="xxx-input"]', '入力値');
      await page.click('[data-testid="submit-button"]');
      // Then
      await expect(page.locator('[data-testid="result"]')).toBeVisible();
    });
  });

  test.describe('異常系', () => {
    test('必須項目が空のときエラーが表示される', async ({ page }) => {
      await page.click('[data-testid="submit-button"]');
      await expect(page.locator('[data-testid="error-message"]')).toBeVisible();
    });
  });

  test.describe('境界値', () => {
    test('[境界値テスト名]', async ({ page }) => {
      // 具体的な境界値を入れる
    });
  });
});
```

## MODE 2: Antigravity — 探索的テスト

### 出力するもの（3点すべて必須）

1. **操作ログ** — 時系列で何をしたかを記録する
2. **再現手順** — 発見した問題を誰でも再現できる手順で書く
3. **重大度** — High / Medium / Low を基準に従って付ける

### 実施する操作（全項目を実際に試す）

**想定外操作**
- ボタンを0.5秒以内に3回連打（二重送信・多重登録の確認）
- ブラウザの「戻る」を操作途中で押す
- フォーム入力中にページをリロードする
- 複数タブで同じ操作を並行して行う

**異常入力（各フィールドに以下を入力して送信する）**
| 入力値 | 確認内容 |
|---|---|
| 空文字 `""` | エラーメッセージが出るか |
| スペースのみ `"   "` | 空文字と同じ扱いか |
| 300文字以上の長文 | UIが崩れないか・制限エラーが出るか |
| `<script>alert('xss')</script>` | スクリプトが実行されないか |
| `'; DROP TABLE users; --` | エラーにならずに処理されるか |
| `🎉🚀💥` 絵文字 | 文字化け・クラッシュしないか |

**UI崩れ確認（ブラウザ幅を変更して確認）**
- 375px（iPhone SE相当）
- 768px（タブレット相当）

### 操作ログの書き方（必須フォーマット）

```
## Antigravity 操作ログ: [機能名]
実施日時: YYYY-MM-DD HH:MM
実施時間: XX分

### 操作記録（時系列）
[HH:MM] http://localhost:3000/xxx を開いた
[HH:MM] [data-testid="submit-button"] を3回連打した
[HH:MM] → 同じデータが3件登録された ← 問題発見
[HH:MM] フィールドに '<script>alert("xss")</script>' を入力して送信
[HH:MM] → スクリプトは実行されなかった。テキストとして表示された ← 正常
[HH:MM] 画面幅を375pxに変更した
[HH:MM] → 「追加」ボタンが画面外にはみ出した ← 問題発見
...
```

### 不具合の記録フォーマット（1件ずつ書く）

```
### バグ #[N]: [概要タイトル（30文字以内）]

重大度: High / Medium / Low
理由: [上記の判定基準に照らした根拠]

発生条件:
- ブラウザ: [名前 + バージョン]
- 画面サイズ: [px]
- ユーザー状態: [ログイン済み / 未ログイン / etc.]

再現手順:
1. http://localhost:3000/[パス] を開く
2. [具体的な操作]
3. [具体的な操作]
4. [何が起きるかを確認する]

期待する動作: [仕様書に基づいたあるべき挙動]
実際の動作: [実際に起きていること（エラーログがあれば貼る）]

Engineerへのコメント: [修正の手がかりになる情報]
```

### 重大度の判定基準

| 重大度 | 条件 |
|---|---|
| **High** | 主要機能が使えない / データ損失 / XSS等のセキュリティ問題 / アプリがクラッシュ |
| **Medium** | 機能は使えるが一部の操作で問題 / エラーメッセージが不正確 / 代替手段がある |
| **Low** | 見た目の軽微な崩れ / 誤字脱字 / 一部環境でのみ発生する軽微な問題 |
