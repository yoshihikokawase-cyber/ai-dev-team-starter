# ターゲットユーザー定義

---

## 主要ターゲット: 個人開発者 / スタートアップエンジニア

### ユーザー像

**名前**: 田中 健（仮名）
**職種**: フルスタックエンジニア（スタートアップ創業メンバー）
**規模**: 3〜5名のエンジニアチーム

### ユーザーの状況・悩み

| 悩み | 詳細 |
|---|---|
| 役割が多すぎる | PM・BA・エンジニア・テスターを1〜2人で兼務している |
| 仕様書を書く時間がない | 「頭の中にある仕様」をそのまま実装して後から困る |
| テストが後回しになる | 実装優先で、テストはほぼ手動確認のみ |
| コードの全体像を把握しづらい | コードが増えてくると、どこに何があるか忘れる |
| バグが本番で発覚する | 探索的テストを行う時間・スキルがない |

### 利用シーン

**シーン1: 新機能の開発**
「新しい機能のアイデアがある。仕様書を書く時間は取れないが、後から揉めたくない。」
→ BAエージェントに要望を渡して仕様書を作ってもらう

**シーン2: 既存コードの修正**
「バグ修正したいが、どこを直せばいいか把握していない。影響範囲が怖い。」
→ Knowledge Researcherで関連コードを調査し、Engineerで最小差分修正する

**シーン3: リリース前の品質確認**
「リリース前にちゃんとテストしたいが、手動テストに時間をかけられない。」
→ Testerエージェントでシナリオテストと探索テストを一気に実施する

## サブターゲット: AIエージェント活用に興味があるソロ開発者

### ユーザー像

**名前**: 鈴木 彩（仮名）
**職種**: フリーランスエンジニア
**スキル**: バックエンド中心、フロントエンドは勉強中

### ユーザーの状況・悩み

| 悩み | 詳細 |
|---|---|
| Claude Codeの使い方が定まらない | 「何でも聞けるAI」として使っており、一貫したフローがない |
| 毎回ゼロから指示する | 同じような指示を毎回書き直している |
| 品質が安定しない | やり取りの質が気分によって変わる |

### 利用シーン

**シーン1: Claude Codeに役割を持たせて使いたい**
「毎回「〜してください」と書くのが面倒。専門ロールとして動かしたい。」
→ agents/ フォルダーの定義ファイルを使い、専門エージェントとして呼び出す

**シーン2: 一貫した出力形式にしたい**
「毎回違う形式で返ってくるので、使いにくい。」
→ 各エージェントの出力形式を定義し、統一されたアウトプットを得る

## このキットが向いていない人

- 大規模チーム（10名以上）で複雑なワークフロー管理ツールが必要な場合
- Jira / Linear / Notionなどのプロジェクト管理ツールとの深い統合が必要な場合
- 完全自律的なCI/CDパイプライン構築が主目的の場合
