# 動作例: Project Manager Agent

## サンプル案件

**テーマ**: シンプルなTodo管理Webアプリを作る

---

## 呼び出し例

```
agents/project-manager.md を読んで、以下のゴールに向けてタスクを分解してください:

ゴール:
「Todoを追加・完了・削除できるシンプルなWebアプリを1週間で作る」

技術スタック:
- フロントエンド: React + TypeScript
- バックエンド: Node.js + Express
- データベース: SQLite（開発用）
- テスト: Playwright

制約:
- エンジニアは1名（AIエージェント）
- 今週中にリリースしたい
- 認証機能は今回は不要

今日やることを3件に絞ってください。
```

---

## ポイント

- **タスクの粒度**: 半日〜1日で完了できるサイズに分解されている
- **依存関係が明確**: 何が終わってから次に進むかが整理されている
- **今日やることが3件**: 多すぎず、少なすぎず
- **完了条件が具体的**: 「仕様書が作成されている」ではなく「〜が記載されている」まで書かれている
