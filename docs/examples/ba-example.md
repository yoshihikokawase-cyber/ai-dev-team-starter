# 動作例: Business Analyst Agent

## サンプル案件

**テーマ**: シンプルなTodo管理Webアプリ — Todo機能の仕様書を作る

---

## 呼び出し例

```
agents/business-analyst.md を読んで、以下の要望を仕様書にしてください:

「Todoを追加・完了・削除できるシンプルなWebアプリを作りたい。
 Todoには「タイトル」と「期限日（任意）」を入力できるようにしたい。
 完了したTodoは見た目を変えて（グレーアウトなど）残しておきたい。
 削除は取り消しできなくていい。
 ユーザー管理・ログインは今回は不要。」

仕様書は docs/requirements/todo-feature.md として保存してください。
```

---

## BAエージェントの出力例（docs/requirements/todo-feature.md）

```markdown
# 仕様書: Todo管理機能

作成日: 2026-03-27
作成者: Business Analyst Agent
ステータス: 承認済み

---

## 背景・課題

タスクを紙やメモアプリで管理しているが、一元管理できていない。
シンプルに「やること」を追加・完了・削除できるWebアプリが欲しい。

## 目的

ユーザーが日々のタスクをブラウザ上で管理できるようにする。
複雑な機能は不要で、すぐに使えるシンプルさを重視する。

## 対象ユーザー

- ユーザー種別: 認証不要（誰でもアクセス可能）
- 利用シーン: 日々の業務タスク・個人タスクの管理

## ユースケース

### UC-01: Todoを追加する
- アクター: ユーザー
- 事前条件: アプリが開かれている
- 基本フロー:
  1. タイトル入力フィールドにテキストを入力する
  2. 期限日を選択する（任意）
  3. 「追加」ボタンをクリックする（またはEnterキーを押す）
  4. TodoリストにTodoが追加される
- 代替フロー:
  - タイトルが空の場合: エラーメッセージを表示し、追加されない
- 事後条件: 入力フィールドが空になる

### UC-02: Todoを完了にする
- アクター: ユーザー
- 事前条件: 未完了のTodoが存在する
- 基本フロー:
  1. Todoの左側のチェックボックスをクリックする
  2. Todoがグレーアウトし、テキストに取り消し線が入る
  3. チェックボックスがチェック済みになる
- 代替フロー:
  - チェック済みのTodoを再クリックした場合: 未完了状態に戻る
- 事後条件: Todoのステータスが「完了」になる

### UC-03: Todoを削除する
- アクター: ユーザー
- 事前条件: Todoが存在する
- 基本フロー:
  1. 削除したいTodoの「削除」ボタンをクリックする
  2. Todoがリストから削除される（確認ダイアログは表示しない）
- 事後条件: TodoがDBから削除される

## 機能要件

- [ ] FR-01: Todoを追加できる（タイトル必須・期限日任意）
- [ ] FR-02: Todoの一覧を表示できる（未完了・完了を区別して表示）
- [ ] FR-03: Todoを完了・未完了に切り替えられる
- [ ] FR-04: Todoを削除できる（確認ダイアログなし）
- [ ] FR-05: タイトルが空の場合、追加できずエラーメッセージを表示する
- [ ] FR-06: Enterキーでもフォームを送信できる
- [ ] FR-07: ページリロード後もTodoが保持される（データ永続化）

## 非機能要件

- パフォーマンス: 100件のTodoが存在しても、表示が2秒以内
- セキュリティ: XSS対策（入力値をHTMLとして解釈しない）
- 対応ブラウザ: Chrome最新版 / Firefox最新版 / Safari最新版
- 対応デバイス: Desktop / Mobile

## 受け入れ条件

- Given アプリが開かれている, When タイトルに「買い物」と入力して「追加」ボタンをクリックする, Then 「買い物」というTodoがリストに追加される
- Given アプリが開かれている, When タイトルを空にして「追加」ボタンをクリックする, Then 「タイトルを入力してください」というエラーが表示され、Todoは追加されない
- Given 未完了のTodoが存在する, When チェックボックスをクリックする, Then Todoがグレーアウトし取り消し線が入る
- Given Todoが存在する, When 「削除」ボタンをクリックする, Then Todoがリストから消える
- Given ページをリロードする, When リロードが完了する, Then 追加したTodoが残っている

## スコープ外（今回含めないもの）

- ユーザー登録・ログイン: 理由: シンプルさを優先するため
- Todoの編集（タイトルの変更）: 理由: 削除して再登録で代替できるため
- Todoの並び替え（ドラッグ&ドロップ）: 理由: 実装コストが高いため
- 複数ユーザーでの共有: 理由: 認証が不要なため

## 未確認事項

- なし（全て要件が明確）
```

---

## ポイント

- **受け入れ条件が Given/When/Then 形式**: テスト可能な具体的な条件になっている
- **スコープ外の理由が明記されている**: 実装者が「これは作らなくていい」と迷わない
- **機能要件がチェックボックス形式**: 実装の進捗を確認しやすい
- **未確認事項がない**: 不明点があれば実装前に解消している
