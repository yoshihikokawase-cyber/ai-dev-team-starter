# MVP要件定義

このドキュメントはAI開発チームスターターキット自体のMVPを定義します。
新しいプロジェクトに使う場合は、このファイルをプロジェクト用に書き換えてください。

---

## MVPの目的

最小限の構成で「AI開発チームが実際に動く」状態を実現する。
エージェントが連携して、要件定義→実装→テストの1サイクルを完走できること。

---

## MVPに含めるもの

### 必須（Must Have）

| # | 機能・成果物 | 完了の定義 |
|---|---|---|
| M1 | 5ロール分のエージェント定義ファイル | 各エージェントを呼び出して意図通りの出力が得られる |
| M2 | CLAUDE.md（共通ルール） | Claude Codeが自動読込し、全エージェントに適用される |
| M3 | 要件定義テンプレート（write-spec） | 要望を渡すと仕様書が生成できる |
| M4 | 実装スキル（implement-feature） | 仕様書をもとに最小差分の実装ができる |
| M5 | Playwrightテストスキル | 正常系・異常系・境界値のテストコードが生成できる |
| M6 | Antigravityテストスキル | 探索的テストの観点と報告書が生成できる |
| M7 | docs/ 知識ベース初期ファイル | プロジェクト情報が参照できる状態 |

### 推奨（Should Have）

| # | 機能・成果物 | 理由 |
|---|---|---|
| S1 | エージェント動作例（examples/） | 初回利用者が使い方を学べる |
| S2 | 不具合報告テンプレート（runbooks/test.md） | バグ記録の一貫性を保つ |
| S3 | 意思決定記録（decisions/） | なぜこの構成にしたかが残る |

### あると良い（Could Have）

| # | 機能・成果物 | 補足 |
|---|---|---|
| C1 | CI/CD設定ファイル（GitHub Actions） | Playwrightテストの自動実行 |
| C2 | eslint / prettier 設定 | コードスタイルの統一 |
| C3 | Notionやスプレッドシートとの連携例 | タスク管理ツールとの統合 |

---

## MVPに含めないもの（スコープ外）

| 除外事項 | 理由 |
|---|---|
| データベース設計・マイグレーション | プロジェクト固有の内容のため、初期テンプレートには含めない |
| 認証・ユーザー管理の実装コード | 同上 |
| Slack / メール通知 | 第2フェーズで追加する |
| 多言語対応（i18n） | 同上 |
| パフォーマンス最適化の自動化 | 同上 |

---

## 優先順位付けの基準

1. **高**: エージェントが動く・連携できる（コア機能）
2. **中**: 品質と再利用性を高める（テンプレート・ドキュメント）
3. **低**: 周辺ツールとの統合・最適化

---

## 次のフェーズ（MVP後）

### Phase 2
- CI/CD（GitHub Actions）でPlaywrightを自動実行
- エージェントの出力をNotionやMarkdownファイルへ自動保存
- 複数プロジェクトへの横展開用テンプレート化

### Phase 3
- エージェント間の自動連携（PMのタスク完了をトリガーにBAを呼び出すなど）
- 過去の意思決定をRAGで参照する仕組み
- パフォーマンス計測・リリース判断の自動化
