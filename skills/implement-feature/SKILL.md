# Skill: 機能を実装する（implement-feature）

## このスキルの目的

仕様書・調査結果をもとに、機能を最小差分で実装します。
Engineerエージェントの行動手順に従い、実装前・中・後のチェックを行います。

---

## 使い方

```
skills/implement-feature/SKILL.md を読んで、
docs/requirements/[仕様書名].md の仕様に従い、[機能名]を実装してください。
```

---

## 実装手順

### STEP 1: 仕様確認（着手前）

```
# 確認すること
1. docs/requirements/[仕様書名].md を読む
2. 受け入れ条件を全て把握する
3. スコープ外を確認する（実装してはいけないものを把握する）
4. 未確認事項がある場合は実装前に確認する
```

### STEP 2: 既存コード調査（着手前）

```
# 調査すること
1. 関連する既存ファイルを読む
2. 命名規則・ディレクトリ構造のパターンを把握する
3. 使用しているライブラリ・フレームワークのバージョンを確認する
4. 変更するファイルを列挙する
5. 影響を受ける既存機能を特定する
```

### STEP 3: 実装計画の作成（着手前）

実装前に以下を明示してください:

```markdown
## 実装計画

### 変更するファイル
- path/to/file1.ts: [変更内容の概要]
- path/to/file2.ts: [変更内容の概要]

### 変更しないファイル（影響範囲確認）
- path/to/other.ts: [変更しないが影響を受けるファイル]

### 実装の順序
1. [最初にやること]
2. [次にやること]
3. [最後にやること]
```

### STEP 4: 実装

```
実装時の注意:
- 既存のコードスタイル・命名規則に合わせる
- ハードコードを避ける（設定値は環境変数や定数ファイルへ）
- エラーハンドリングを忘れない
- 型定義を正しく行う（TypeScriptの場合）
- セキュリティ上の問題がないか確認する
  - XSS: ユーザー入力をそのままHTMLに挿入しない
  - SQLインジェクション: パラメータ化クエリを使う
  - 認証・認可: 適切なチェックを実装する
```

### STEP 5: 検証（実装後）

```bash
# ビルド確認
npm run build

# テスト実行
npm test

# 型チェック（TypeScriptの場合）
npm run type-check

# リント
npm run lint
```

### STEP 6: 実装レポート作成

```markdown
## 実装レポート: [機能名]

### 1. 方針
[何をどう実装したか]

### 2. 変更ファイル一覧
| ファイルパス | 変更内容 |
|---|---|
| path/to/file.ts | [変更内容] |

### 3. 主要な実装内容
[重要な変更点のみ箇条書き]

### 4. 検証結果
- ビルド: ✅ 成功
- テスト: ✅ 全通過（X件）
- 手動確認: ✅ 動作確認済み

### 5. 残課題
- [課題1] — 優先度: Medium
```

---

## よくあるミスと対策

| ミス | 対策 |
|---|---|
| 既存機能を壊した | 実装前に既存テストを全て実行し、通過を確認する |
| 仕様外の機能を実装した | 仕様書のスコープ外を再確認する |
| テストがない状態でリリースした | 実装と同時にテストコードを書く |
| ハードコードが残った | `grep -r "localhost" src/` などで確認する |
| エラーハンドリングを忘れた | try-catch / エラーレスポンスの処理を実装する |
