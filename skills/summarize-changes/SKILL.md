# Skill: 変更内容を要約する（summarize-changes）

## このスキルの目的

実装した変更内容を、チームメンバーやレビュアーが理解できる形で要約します。
PR説明文・リリースノート・振り返り資料の作成に使います。

---

## 使い方

```
skills/summarize-changes/SKILL.md を読んで、
以下の変更内容を要約してください:
[変更内容または git diff の出力]

出力形式: PR説明文 / リリースノート / 振り返り資料
```

---

## 出力形式1: PR説明文

```markdown
## 概要

[1〜3文で何をしたかを説明]

## 変更内容

### 追加した機能
- [追加した機能1]
- [追加した機能2]

### 変更した機能
- [変更した機能1]: [変更前] → [変更後]

### 修正したバグ
- [修正したバグ1]: [原因と修正内容]

## 変更したファイル

| ファイルパス | 変更種別 | 変更内容 |
|---|---|---|
| path/to/file.ts | 追加 | [内容] |
| path/to/file.ts | 変更 | [内容] |
| path/to/file.ts | 削除 | [内容] |

## テスト

- [ ] 既存テストが全て通過している
- [ ] 新機能のテストを追加した（X件）
- [ ] 手動での動作確認が完了している

## 確認してほしい点

- [レビュアーに特に確認してほしい箇所]
- [判断が必要な設計上の決定]

## 関連リンク

- 仕様書: docs/requirements/[仕様書名].md
- 関連Issue / チケット: [#番号]
```

---

## 出力形式2: リリースノート

```markdown
## v[バージョン番号] — YYYY-MM-DD

### 新機能
- **[機能名]**: [ユーザー向けの説明]

### 改善
- **[改善内容]**: [何がどう変わったか]

### バグ修正
- **[バグの概要]**: [修正内容]

### 破壊的変更（Breaking Changes）
- なし / [変更内容と移行方法]

### アップグレード手順
```bash
# 必要な場合のみ記載
npm install
npm run migrate
```
```

---

## 出力形式3: 振り返り資料（スプリント振り返り）

```markdown
## 振り返り: [スプリント名 / 期間]

### 完了したこと
- [タスク1] — 担当: [エージェント名]
- [タスク2] — 担当: [エージェント名]

### 完了しなかったこと（持ち越し）
- [タスク名] — 理由: [なぜ完了しなかったか]

### 発見した問題・学び
- [問題または学び1]
- [問題または学び2]

### 次のスプリントに向けて
- [改善アクション1]
- [改善アクション2]

### 数値サマリー
- 完了タスク数: X件
- 発見バグ数: X件（High: X / Medium: X / Low: X）
- 解消バグ数: X件
```

---

## まとめ時の注意点

### 良い要約の条件
- **事実ベース**: 「改善した」ではなく「レスポンスタイムをX秒→Y秒に短縮した」
- **影響範囲が明確**: 何が変わり、何が変わっていないかを明示する
- **次のアクションがわかる**: 読んだ人が何をすべきかがわかる

### 避けること
- 実装の詳細すぎる説明（コードレビューのコメントに任せる）
- 「やった」「直した」のみの記述（何を・なぜを書く）
- 推測や意図を事実として書く
