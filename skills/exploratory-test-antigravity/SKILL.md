# Skill: Google Antigravityで探索的テストを行う（exploratory-test-antigravity）

## このスキルの目的

Google Antigravityを使い、Playwrightで検出できない「想定外の操作」「UI崩れ」
「データ不整合」「エッジケース」を探索的に発見します。

---

## 使い方

```
skills/exploratory-test-antigravity/SKILL.md を読んで、
[機能名 / 対象URL] に対して探索的テストを実施してください。

実施時間の目安: 30〜60分
対象ページ: [URL]
仕様書: docs/requirements/[仕様書名].md
```

### 目的の確認
- [ ] テスト対象の仕様書を読んだ

## 探索テストの実施観点

### 観点1: 想定外の操作

**ランダム操作**
- 画面上のボタンをランダムな順序でクリックする
- フォームフィールドをランダムな順序で入力する
- 意図しないタイミングでページを離れる

**高速操作**
- 送信ボタンを素早く連打する（二重送信）
- スクロールを高速で繰り返す
- タブキーで高速にフォーカスを移動する
- ブラウザの戻る・進むを素早く繰り返す

**複数タブ操作**
- 同じページを複数タブで開いて同時に操作する
- 一方のタブで操作してから他方のタブをリロードする

---

### 観点2: 異常入力

以下の入力パターンをフォームフィールドに試みる:

```
# 空・スペース系
""（空文字）
" "（スペースのみ）
"　"（全角スペースのみ）

# 超長文
"a" × 1000文字
"あ" × 500文字
URLを1000文字貼り付け

# 特殊文字・記号
"<script>alert('xss')</script>"
"'; DROP TABLE users; --"
"../../../etc/passwd"
"${7*7}"（テンプレートインジェクション）
"\n\t\r"（制御文字）
"🎉🚀💥"（絵文字）
"ÄÖÜäöü"（ウムラウト等）
"中文日本語한국어"（多言語）
"ＡＢＣ"（全角英数）
"０１２"（全角数字）

# 数値フィールド向け
0
-1
-999999
9999999
3.14159
"abc"（文字列）
Infinity / NaN
```

---

### 観点3: UI崩れ確認

**画面サイズ別確認**

| サイズ | 幅 | 確認内容 |
|---|---|---|
| モバイル（iPhone SE） | 375px | ボタンが隠れていないか / テキストが見切れていないか |
| モバイル（大きめ） | 430px | レイアウト崩れがないか |
| タブレット | 768px | 横並びと縦並びの切り替わりが正しいか |
| ラップトップ | 1280px | 標準的なデスクトップ表示 |
| 大画面 | 1920px | 横幅いっぱいに崩れていないか |
| 4K | 2560px | 画像・フォントの拡大が正しいか |

**その他のUI確認**
- [ ] テキストズーム200%でレイアウトが崩れないか
- [ ] ダークモードで文字・アイコンが見えるか
- [ ] 高コントラストモードで使えるか
- [ ] アニメーション・ローディング中の見た目
- [ ] エラー状態の見た目（赤文字・アイコン等）

---

### 観点4: データ不整合確認

- [ ] フォーム入力中にページをリロードした場合
- [ ] フォーム入力中にブラウザを閉じた場合
- [ ] 同じデータを二重に登録した場合
- [ ] 削除したデータを参照しようとした場合
- [ ] 他のユーザーが編集中のデータを同時に編集した場合
- [ ] ネットワーク切断後に操作した場合（オフライン挙動）

---

## エラー記録方法

### 即時記録
バグを発見したら、その場で以下をキャプチャする:
1. スクリーンショット（または画面録画）
2. ブラウザのコンソールエラー（F12 → Console）
3. ネットワークエラー（F12 → Network）
4. 再現手順（頭の中にある間に書き留める）

### バグ報告書の作成

```markdown
### バグ報告 #[番号]: [概要タイトル]

**重大度**: High / Medium / Low

**発生条件**
- ブラウザ: [名前 + バージョン]
- OS: [名前 + バージョン]
- 画面サイズ: [幅 × 高さ px]
- ユーザー状態: [ログイン済み / 未ログイン / 特定の状態]

**再現手順**（誰でも再現できるレベルで書く）
1. [URLを開く]
2. [操作1]
3. [操作2]
4. [確認する]

**期待する動作**
[本来こうなるべき挙動]

**実際の動作**
[実際に起きていること]

**スクリーンショット**
[ファイル添付またはパス]

**コンソールエラー**
```
[エラーログを貼り付け]
```
```

---

## 重大度の判定基準

| 重大度 | 条件 | 例 |
|---|---|---|
| **High** | 主要機能が使えない / データ損失 / セキュリティ問題 | 登録ボタンが反応しない / XSSが通る |
| **Medium** | 機能は使えるが一部で問題 / 代替手段がある | エラーメッセージが不正確 / UIが一部崩れる |
| **Low** | 軽微な見た目の問題 / 誤字脱字 | ボタンの余白がおかしい / 英語のエラーが混在 |
